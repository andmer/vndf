Welcome

Hello, and welcome to this month's Von Neumann Defense Force newsletter! I'm Hanno, VNDF's lead developer and chief newsletter engineer.

As the title says, this has been a rather uneventful month. I've recently picked up a new client (I'm a freelancer) and am still adjusting to my new schedule. Still, progress on VNDF hasn't halted fully. Here's what's been going on this month:

  • New Newsletter Format: As part of my ongoing quest to make me finishing this game a realistic prospect, I've decided to cut back the time spent on the newsletter.
  • Logging Infrastructure: I've been hunting for a persistent bug that's been eluding me for a while. Better logging should help me out.
  • Small Fixes: A few small annoyances have been fixed.

Have fun with this month's newsletter and please email any feedback to mail@hannobraun.de. Do you have any questions about VNDF in general or the contents of this newsletter specifically? Let me know!

New Newsletter Format

Ensuring a successful outcome for Von Neumann Defense Force, despite the amount of work it requires and the limited time I can spend on it these days, has been an ongoing concern for me. Those last few months, I've been cutting down on work where I can, and now it's time to question how I can reduce the monthly work of getting this newsletter out.

A few months ago, I already announced that I wasn't going to write a new article for every newsletter any more, since that had been the most difficult and time-consuming thing. From today on, I'm cutting out the articles for good. In addition, the news section at the end is also gone. That only leaves the most important thing: The development update, to let everyone know what's going on month to month.

Whether you like this change or not, please let me know what you think!

Logging Infrastructure

One thing that's been ongoing for a while now is the hunt for a very persistent bug: From time to time, I was logging into my server to find that the VNDF server process was running completely wild, using 100% CPU. It's only been happening very rarely, and thus has been difficult to diagnose.

In order to find this bug and other similar ones in the future, I've been introducing a proper logging infrastructure using log. What this means is, the server process will now print out messages for almost everything it does. Since different log levels are used, it's quite easy to filter out everything non-essential or look at all the details, depending on what's required.

So far it hasn't helped me with my bug yet, but it's good to have around none the less.

Small Fixes

With the introduction of the new hybrid interface came a few new problems. Among those: Some key presses (Backspace and Return) were registered twice and players no longer had their own comm id displayed to them.

The first problem was a straight-forward bug fix. The second was solved by introducing a new command: Players can now see their own comm id with the new comm-data command.
